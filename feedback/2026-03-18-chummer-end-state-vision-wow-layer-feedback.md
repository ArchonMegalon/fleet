# Chummer End-State Vision Feedback

Date: 2026-03-18

This is a design-direction packet for the existing Chummer horizons and LTD-canon work. Treat it as north-star canon guidance, not as new release-blocking scope.

## Verdict

`chummer6-design` is ready to raise the ambition of the end-state now, because the structural prerequisites already exist:

- canonical `HORIZONS.md`
- machine-readable `HORIZON_REGISTRY.yaml`
- public-guide export manifest
- LTD capability map
- external-tools plane with newer tools already inventoried

Keep release sequencing foundation-first. Fold this into the non-blocking Horizons / external-tools canon track, not into current `vnext-foundation` blockers.

## Main change in direction

Evolve the north star from:

- trustworthy deterministic engine
- workbench and session OS
- hosted orchestration
- explainable artifact/media surfaces

to:

- truth engine
- session OS
- runsite explorer
- artifact studio
- creator press
- explainable replay / forensics ecosystem

The key rule:

> End-state wow may outrun current release scope, but it must not overrule foundation sequencing.

## Pillars to make explicit in design canon

### 1. Truth Engine

Keep deterministic, explainable, provenance-bearing runtime truth as the bedrock.

### 2. Session OS

Sharpen the lived promise:

- reconnect without panic
- replay missed state
- settle rules disputes with receipts
- keep the table moving

### 3. Runsite Explorer

Promote bounded explorable mission/location packs to an explicit end-state pillar:

- tours
- hotspots
- floor plans
- route overlays
- optional narration

Boundaries:

- not live session truth
- not combat truth
- not a VTT replacement

### 4. Artifact Studio

Make `JACKPOINT` more central:

- dossiers
- recaps
- narrated briefings
- share cards
- creator packs
- evidence-bearing artifacts

### 5. Creator Press

Promote `RUNBOOK PRESS` as the long-form publishing lane:

- primers
- faction books
- district guides
- campaign books
- convention modules

### 6. Replay / Forensics

Open a new horizon:

- session replay
- after-action review
- receipts-over-time
- grounded recap artifacts
- branchable what-if comparisons

Suggested horizon key: `ghostwire`

### 7. Advisory Community Signal

Allow public idea/signal intake as an advisory horizon family, but keep it explicitly non-canonical and non-runtime-truth.

## Horizon refinements

### `runsite`

Strengthen it as the mission-space explorer for briefing, planning, and spatial understanding.

### `jackpoint`

Define it as the short/medium-form artifact studio:

- dossiers
- recaps
- narrated briefings
- evidence rooms
- social/share outputs

### `runbook-press`

Define it as the long-form publishing lane:

- primers
- handbooks
- city books
- campaign books
- convention modules

### `ghostwire`

Add it as a new horizon for replay / forensics / explainable recap.

## Risks to guard against

### 1. Tool-led architecture

Do not let owned LTD inventory dictate the product.

### 2. Hub bloat

Do not let new wow lanes silently collapse into `chummer6-hub`.

### 3. Media becoming truth

Tours, videos, books, dossiers, and recaps must remain derivative artifacts, not canonical runtime truth.

### 4. Horizon inflation before foundation completion

Keep this north-star work non-blocking until current foundation blockers are actually closed.

### 5. Guide outrunning canon again

The public guide must remain downstream from design horizon registry and export manifests.

### 6. No promotion path from horizon to build lane

Every horizon needs explicit owner handoff and success criteria before it becomes active build scope.

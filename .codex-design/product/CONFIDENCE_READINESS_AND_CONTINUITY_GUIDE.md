# Confidence, Readiness, and Continuity

## Product sentence

Chummer6 keeps Shadowrun understandable when the game gets complicated.

That is the sharper user-facing center of gravity.
The architecture can remain broad, but the product promise needs to stay emotionally simple:

* confidence: my numbers are right, and I can see why
* readiness: I know what to do next at the table
* continuity: the campaign remembers what happened

## What this changes

Chummer6 should not be led as "everything for Shadowrun."
It should be led as the explainable campaign OS that makes a crunchy game feel calm under pressure.

The acquisition wedge is still build and explain.
The retention engine is campaign continuity.
The long-term moat is trust plus portability plus memory, not feature-count sprawl.

## Four product surfaces

### 1. Chummer Build

The trustworthy runner-building and advancement surface.

Includes:

* quickbuild
* expert workbench
* explain drawer
* legality and warning posture
* manual intake
* migration review
* advancement planning
* gear, drone, deck, and cyberware labs

### 2. Chummer Play

The at-table player and GM shell.

Includes:

* mobile table sheet
* condition, edge, ammo, and active-effect state
* quick explain
* roll receipts
* session log
* rejoin and resume
* player-safe recap handoff

### 3. Chummer Campaign

The campaign-memory and GM-operations surface.

Includes:

* roster
* rule environment
* contacts
* jobs and legwork
* debts, favors, and rewards
* downtime and aftermath
* recap history
* open disputes
* next-session cockpit

### 4. Chummer Exchange

The portability and proof surface.

Includes:

* portable dossier
* campaign exchange
* Foundry-class export/import
* PDF packets
* support packet
* release manifest and proof shelf
* ruleset fingerprint and compatibility receipts

## Habit loop we want

The good repeat-use loop is not compulsion.
It is continuity.

Users should return because:

* the campaign lives here
* the product remembers what the table would otherwise forget
* prep and play feel lighter
* recovery is calmer than ad hoc notes

## Implementation bets

### 1. Universal Explain Drawer

Every important value should answer:

* what is this value
* why is it this value
* what changed it
* which source or rule environment contributed to it
* what warnings or conflicts exist
* what would make the warning go away

It should also support:

* copy explanation
* share with GM
* mark disputed
* attach to campaign ruling

### 2. Manual Intake Mode

Users need a serious "I already have a runner" path.

The flow should support:

* start from current truth
* mark unknown history explicitly
* enter confirmed values first
* reconcile provenance later

### 3. Migration Confidence

Every import or manual-adoption result should emit:

* migration confidence
* what was confirmed
* what still needs review
* safe-to-play posture
* bounded-loss or warning receipts

### 4. Quickbuild plus Expert Workbench

Do not force a false choice between beginner guidance and dense expert tooling.

Chummer Build should support:

* concept-first quickbuild
* smart defaults
* reversible steps
* direct drop into dense expert workbench without losing provenance

### 5. Mobile Table Sheet first

Mobile should start as the indispensable play surface, not a compressed desktop builder.

First value:

* current sheet
* condition
* edge
* initiative
* ammo and gear
* spells, powers, and programs
* quick explain
* rejoin and resume

### 6. GM Next Session cockpit

A GM should have one obvious page for:

* what happened last session
* unresolved hooks
* who is injured
* which rewards are pending
* open rules disputes
* contacts, favors, and heat
* prep tasks and next scenes

### 7. Rule Environment Studio

Every campaign should expose one governed rule environment:

* edition and sources
* bans and caps
* optional rules
* house rules
* custom data packages
* GM rulings
* version history

Every validation result should state which environment it is valid under.

### 8. Run Recap Cards

After each run, Chummer should emit an editable compact recap with:

* outcome
* rewards
* open threads
* world or campaign consequences
* player-safe and GM-private variants

### 9. Public proof shelf

Trust should be visible on the public surface.

For each release:

* artifact manifest
* checksums
* smoke result
* known issues
* migration posture
* support packet example
* release notes written for users

### 10. One strong ecosystem bridge

Chummer should not try to replace every play surface.
It should dominate truth and interoperate aggressively.

The first bridge should prove:

* portable dossier
* one first-class external exchange target
* support and provenance receipts surviving export

### 11. Campaign adoption mode

Existing tables need "adopt my current campaign," not only "start fresh."

That flow should support:

* current runners
* current house rules
* current contacts, debts, and jobs
* unknown historical provenance
* safe continuation from today

### 12. Career and continuity loops

Make progress satisfying without manipulative gameification.

Candidate loops:

* runner career timeline
* karma forecast
* nuyen envelopes
* crew fit meter
* legwork board
* contact relationship ledger
* campaign health check

## What to de-emphasize until the core loop is loved

These are still valid horizons, but they should not outrun the core product story:

* Table Pulse
* large social or leaderboard mechanics
* heavy assistant branding
* media-first launch stories
* creator-press and broader publication breadth
* visible Fleet/control-plane complexity in user-facing surfaces

## Sequencing rule

The next product proof should favor this order:

1. explain drawer
2. manual intake and migration confidence
3. mobile table sheet
4. GM next-session cockpit
5. rule environment studio
6. recap cards
7. public proof shelf
8. first exchange bridge

That is the shortest route from "architecturally impressive" to "emotionally indispensable."

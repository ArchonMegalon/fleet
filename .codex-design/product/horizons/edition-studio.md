# EDITION STUDIO

## The problem

One generic shell can technically support SR4, SR5, and SR6 while still making all three feel flattened.
That is table pain: the user loses comprehension, atmosphere, and confidence exactly where the ruleset should feel authored.

## Bounded product move

EDITION STUDIO would not split Chummer into three apps.
It would give each promoted ruleset a deliberately authored head:

* distinct terminology, prompts, and inspector posture where semantics diverge
* ruleset-specific interaction patterns where a shared workflow becomes confusing or lossy
* visual language, motion, density, and emphasis that reflect each edition's mental model without fragmenting the product

This is an authored product move, not decorative theming.
The point is to preserve meaning through shell-level expression while keeping engine truth and shared workflows intact.

## Likely owners

* `chummer6-ui`
* `chummer6-ui-kit`
* `chummer6-core`

## LTD / tool posture

AI design aids may assist exploration, but they do not author canonical edition posture.
Canonical edition expression stays in the design system and product heads, and rules truth stays downstream of core semantics rather than styling.

## Dependency foundations

* `edition_expression_substrate` - ruleset-specific terminology, composition seams, inspectors, and theming without flattening SR4, SR5, and SR6
* `expert_speed_substrate` - explicit latency budgets and dense-state performance so authored heads do not become slow theater
* `evaluation_substrate` - import oracles, user-facing acceptance gates, and proof surfaces that can show the editions still behave differently

## Current state

EDITION STUDIO is still a horizon.
The current shell does not yet prove stable edition seams strongly enough to support separately authored heads without decorative drift or semantic contradiction.

## Eventual build path

1. Make the edition seams in core and explain explicit and stable.
2. Keep the shared shell primitives reusable, but let each edition compose its own authored head on top.
3. Add edition-specific typography, motion, density, and inspector posture only after the semantics are already reliable.
4. Prove in the release shell that SR4, SR5, and SR6 still read as distinct authored heads instead of one generic facade.
5. Promote the head through the flagship acceptance path only after the proof is repeatable.

## Hard boundary

* not three disconnected apps
* not decorative theming without semantic payoff
* not ruleset flavor that contradicts engine truth

## Flagship handoff gate

The handoff to flagship work happens only when the shell can demonstrate all of the following in a repeatable review:

* SR4, SR5, and SR6 each have a distinct authored head
* the important edition differences remain visible in terminology, inspector posture, and workflow
* the same shared engine truth drives all three heads without forked behavior
* the release shell passes the relevant flagship acceptance evidence, not just a visual mock review

## Why it is still a horizon

Distinct ruleset heads only matter once the shared shell and semantics are already stable.
Until Chummer can prove strong authored SR4, SR5, and SR6 seams, EDITION STUDIO remains a future flagship multiplier rather than live scope.
